## 1. 문서 목적

본 문서는 지역별 토지 정보 플랫폼의 신규 사용자 유입을 위한 랜딩페이지 기획 및 디자인 요구사항을 정의합니다. 랜딩페이지의 목표는 방문자가 "흩어진 토지 정보를 한곳에서 조회할 수 있는 서비스"라는 핵심 가치를 빠르게 이해하고, 지역 검색 체험 또는 앱 설치·사전 신청으로 전환하도록 돕는 것입니다.

## 2. 제품 개요

### 서비스 콘셉트

지역별 토지 정보, 실거래가, 공시지가, 매물 가격, 용도지역·규제 정보를 한 화면에 정리해 보여주는 데이터 조회 서비스입니다. 투자 판단을 대신하거나 특정 토지를 추천하는 서비스가 아니라, 누구나 관심 지역의 땅 정보를 쉽게 찾아볼 수 있게 하는 정보 플랫폼입니다.

### 핵심 가치 제안

- 시·군·구, 읍·면·동 단위로 지역별 토지 정보를 탐색합니다.
- 토지 실거래가와 공시지가, 가격 흐름을 한곳에서 확인합니다.
- 용도지역, 지목, 면적, 도로 접면 등 필지 기본 정보를 쉬운 언어로 요약합니다.
- 여러 사이트에 흩어진 공공 데이터를 지도 기반으로 한눈에 보여줍니다.

### 데이터 제공 범위 (단계 구분)

- 1단계 (초기 출시): 실거래가, 공시지가, 용도지역·지목·면적, 토지이용계획·규제 정보 — 공공 데이터 기반
- 2단계 (추후 검토): 매물 호가 정보 — 공공 API가 없어 확보 방식(중개사 제휴, 직접 등록 등) 확정 후 반영. 확보 전에는 랜딩페이지에서 매물 가격을 제공 기능으로 표기하지 않습니다.

### 톤 앤 매너

- 신뢰감 있는 데이터 서비스, 공공 정보를 정돈해주는 도구의 인상
- 투자 권유·수익 암시가 아니라 정보 접근성 개선을 강조
- 어렵고 딱딱한 토지·행정 용어를 쉽고 정돈된 언어로 설명
- "투자 앱"이 아니라 "토지 정보를 정리해 보여주는 조회 서비스"로 포지셔닝

## 3. 랜딩페이지 목표

### 비즈니스 목표

- 지역 검색 체험, 앱 다운로드, 사전 신청 전환율 증대
- 토지 정보에 관심 있는 잠재 사용자의 리드 확보
- 데이터 출처와 정리 방식에 대한 신뢰도 전달
- "토지 정보는 여기서 본다"는 카테고리 인식 형성

### 사용자 목표

- 이 서비스가 어떤 정보를 보여주는지 빠르게 이해합니다.
- 내 관심 지역의 땅값·실거래를 실제로 조회할 수 있는지 확인합니다.
- 기존에 여러 사이트를 오가던 조사 과정이 어떻게 쉬워지는지 파악합니다.
- 데이터 출처가 믿을 만한지 확인합니다.

### 핵심 CTA

- 1차 CTA: 우리 동네 땅값 확인하기 (웹 체험형 — 지역 검색으로 바로 진입)
- 2차 CTA: 앱 다운로드 또는 출시 알림 신청
- 보조 CTA: 주요 기능 살펴보기

## 4. 타깃 사용자

### 1차 타깃

- 소유하거나 상속받은 토지의 시세·정보를 확인하고 싶은 토지 소유주
- 토지 매수 또는 매도를 검토 중이며 주변 실거래와 시세를 비교하고 싶은 사용자
- 귀농·귀촌, 전원주택, 창고·주차장 등 실사용 목적으로 땅을 알아보는 사용자

### 2차 타깃

- 공인중개사, 토지 컨설턴트 등 지역 토지 정보를 반복 조회하는 전문가
- 지역 개발 정보와 지가 흐름을 꾸준히 모니터링하는 사용자
- 토지에 관심을 갖기 시작해 기초 정보부터 알아보려는 입문자

### 사용자 페인포인트

- 실거래가, 공시지가, 토지이용계획이 각기 다른 사이트에 흩어져 있어 확인이 번거롭습니다.
- 용도지역, 지목, 도로 접면, 행위 제한 등 전문 용어가 어렵습니다.
- 아파트와 달리 토지는 시세를 한눈에 보여주는 서비스가 마땅히 없습니다.
- 내 땅 또는 관심 지역 땅의 가격이 적정한지 주변과 비교하기 어렵습니다.
- 공공 사이트는 정보는 있지만 찾기 어렵고 읽기 어렵습니다.

## 5. 핵심 메시지

### 메인 카피 방향

우리 동네 땅 정보, 한 화면에서 확인하세요.

### 서브 카피 방향

지역별 실거래가, 공시지가, 용도지역, 규제 정보까지 — 여러 사이트를 오가지 않고 한곳에서 조회하세요.

### 메시지 원칙

- 수익 보장, 급등, 확정 호재 등 투자 권유형 표현을 사용하지 않습니다.
- 정보 통합, 조회 편의, 비교 확인, 쉬운 설명을 강조합니다.
- 초보자도 이해할 수 있는 쉬운 문장으로 작성합니다.
- 정보의 출처와 최신성 한계는 명확하게 고지합니다.
- 확보가 확정되지 않은 데이터(매물 호가 등)는 제공 기능으로 표기하지 않습니다.

## 6. 페이지 구조

### 1. Hero 섹션

목적: 첫 화면에서 서비스의 정체성과 체험 행동을 명확히 전달합니다.

구성 요소:

- H1: 우리 동네 땅 정보, 한 화면에서 확인하세요
- 서브 문구: 실거래가부터 공시지가, 용도지역, 규제 정보까지 지역별로 정리해 보여드립니다.
- 1차 CTA: 지역 검색창 또는 "우리 동네 땅값 확인하기" 버튼
- 보조 CTA: 앱 다운로드 / 출시 알림 신청
- 비주얼: 지도 위에 필지·실거래 정보 패널이 열려 있는 땅뷰 화면 목업
- 보조 신뢰 문구: 공공 데이터 기반 정보 조회 서비스임을 명시
- 로고·서비스명: 좌측 상단에 "땅뷰" 로고 배치 (지도·필지 모티브 심볼 검토)

디자인 요구사항:

- 첫 화면에서 지도 기반 서비스 화면이 명확히 보여야 합니다.
- 검색창 또는 체험 CTA가 시각적 중심이 됩니다.
- 배경은 차분한 밝은 톤을 사용하고, CTA는 명확한 대비를 줍니다.
- 부동산 광고 느낌의 금색, 빨간색, 번쩍이는 효과는 피합니다.

### 2. 문제 제기 섹션

목적: 기존 토지 정보 조사 과정의 불편함을 공감시킵니다.

콘텐츠 예시:

- 실거래가 따로, 공시지가 따로, 규제 확인 따로 — 사이트를 몇 번씩 옮겨 다녀야 합니다.
- 아파트 시세 앱은 많은데, 땅 정보를 한눈에 보여주는 곳은 없습니다.
- 용도지역, 지목, 행위 제한 같은 용어가 어렵습니다.
- 내 땅이 얼마인지, 옆 땅은 얼마에 거래됐는지 알기 어렵습니다.

디자인 요구사항:

- 3~4개의 문제 카드를 사용합니다.
- 각 카드는 짧은 제목, 한 줄 설명, 간단한 아이콘으로 구성합니다.
- 카드 내부 텍스트는 모바일에서도 한눈에 읽히도록 짧게 유지합니다.

### 3. 솔루션 섹션

목적: 서비스가 문제를 어떻게 해결하는지 핵심 기능 중심으로 설명합니다.

핵심 기능:

- 지역별 토지 탐색: 시·군·구 → 읍·면·동 단위 브라우징과 지도 탐색
- 실거래가 조회: 주변 토지의 실제 거래 내역과 단가(㎡·평당) 확인
- 공시지가 및 가격 흐름: 연도별 공시지가 변화 확인
- 필지 기본 정보 요약: 용도지역, 지목, 면적, 도로 접면을 쉬운 설명과 함께 제공
- 토지이용계획·규제 확인: 개발 제한, 행위 제한 등 규제 정보 요약
- 관심 지역·필지 저장 및 비교

디자인 요구사항:

- 기능 설명은 실제 서비스 화면 목업과 함께 보여줍니다.
- "무엇을 조회할 수 있는가"와 "어떻게 쉬워지는가"를 중심으로 작성합니다.
- 지도, 필지 정보 패널, 실거래 목록, 비교 화면 등 실제 사용 장면을 보여주는 시각 자료를 우선합니다.
- 매물 호가 기능은 확보 확정 전까지 화면·문구에 포함하지 않습니다.

### 4. 데이터 신뢰 섹션

목적: 방문자가 정보의 출처와 정리 방식에 신뢰를 갖도록 합니다.

콘텐츠 방향:

- 실거래가, 공시지가, 토지이용계획 등 공공 데이터 기반임을 설명합니다.
- 데이터 업데이트 주기와 출처 표기 원칙을 안내합니다.
- 정보의 최신성 한계와 사용자의 최종 확인 필요성을 명시합니다.

데이터 소스 후보 (내부 참고용 — 연동 확정 후 기관명 표기):

- 실거래가: 국토교통부 실거래가 공개시스템 (공공데이터포털 API)
- 공시지가: 부동산 공시가격 알리미
- 토지이용계획·규제: 토지이음(토지이용규제정보서비스)
- 지도·필지 경계: V-World 등 국가공간정보 서비스

주의:

- 실제 연동 데이터가 확정되지 않았다면 구체 기관명은 확정 후 작성합니다.
- "100% 정확", "실시간" 같은 표현은 사용하지 않습니다.

### 5. 정보 체크 섹션

목적: 토지를 볼 때 함께 확인해야 할 항목을 서비스가 빠짐없이 보여준다는 인상을 줍니다.

콘텐츠 예시:

- 용도지역과 행위 제한
- 개발 제한 여부
- 도로 접면과 접근성
- 주변 실거래와의 가격 비교
- 지목과 실제 이용 현황
- 토지이용계획 확인 필요 항목

디자인 요구사항:

- 경고성 빨간색보다 차분한 안내 색상을 사용합니다.
- "확인 완료", "확인 필요", "비교 필요" 등 단계형 상태 표현을 사용합니다.
- 불안감을 조장하기보다 꼼꼼한 확인을 돕는 체크리스트 느낌으로 표현합니다.

### 6. 사용 흐름 섹션

목적: 서비스 사용 과정을 3단계로 단순하게 이해시킵니다.

사용자 흐름:

1. 관심 지역을 검색하거나 지도에서 선택합니다.
2. 필지 정보, 실거래가, 공시지가, 규제 정보를 한 화면에서 확인합니다.
3. 관심 지역과 필지를 저장하고 주변과 비교합니다.

디자인 요구사항:

- 3단계 가로형 또는 세로형 타임라인으로 표현합니다.
- 모바일에서는 세로 스텝 형태로 전환합니다.
- 각 단계마다 작은 서비스 화면 또는 아이콘을 배치합니다.

### 7. 주요 사용자별 활용 사례

목적: 타깃별로 서비스의 활용 상황을 구체화합니다.

활용 사례:

- 토지 소유주: 내 땅의 공시지가 변화와 주변 실거래를 확인
- 매수·매도 검토자: 관심 지역 실거래와 시세를 비교해 적정 가격 감 잡기
- 귀농·전원주택 준비자: 후보지의 용도지역과 규제를 미리 확인
- 중개사·전문가: 여러 필지를 빠르게 조회하고 상담 자료로 활용

디자인 요구사항:

- 페르소나별 카드 3~4개로 구성합니다.
- 사용자 상황, 필요한 정보, 서비스가 주는 도움을 짧게 연결합니다.

### 8. 신뢰 장치 섹션

목적: 데이터 서비스에 필요한 안정감과 투명성을 제공합니다.

구성 요소:

- 데이터 출처 및 업데이트 정책
- 개인정보 보호 안내
- 정보 이용 유의사항 (참고용 정보, 최종 확인 필요)
- 고객 지원 또는 문의 채널
- 베타 서비스라면 베타 한계 고지

디자인 요구사항:

- 지나치게 긴 약관형 문장 대신 핵심 신뢰 요소를 요약합니다.
- 상세 내용은 푸터 또는 별도 링크로 연결합니다.

### 9. 최종 CTA 섹션

목적: 페이지를 읽은 사용자의 전환을 유도합니다.

구성 요소:

- 헤드라인: 지금 땅뷰에서 관심 지역의 땅 정보를 확인해보세요
- 1차 CTA: 지역 검색 체험 또는 앱 다운로드
- 보조 CTA: 출시 알림 받기
- 짧은 유의 문구: 땅뷰는 토지 관련 정보를 정리해 제공하는 조회 서비스이며, 제공 정보는 참고 자료입니다.

## 12. FAQ 초안

### 어떤 정보를 볼 수 있나요?

지역별 토지 실거래가, 공시지가, 용도지역·지목·면적 등 필지 기본 정보, 토지이용계획과 규제 정보를 확인할 수 있습니다.

### 데이터는 어디서 가져오나요?

국토교통부 실거래가 공개 정보 등 공공 데이터를 기반으로 합니다. 각 정보에는 출처와 기준 시점을 표기합니다. (연동 확정 후 구체 기관명 기재)

### 정보는 얼마나 자주 업데이트되나요?

데이터별 갱신 주기가 다릅니다. 각 정보에 기준 시점을 함께 표기하며, 중요한 의사결정 전에는 공부서류와 현장 확인 등을 통한 최종 확인이 필요합니다.

### 매물도 볼 수 있나요?

현재는 실거래·공시지가 등 공공 데이터 기반 정보를 제공합니다. 매물 정보는 준비 중이며, 제공이 확정되면 안내드리겠습니다. (호가 데이터 확보 확정 전까지 이 문항은 노출 여부를 검토)

### 초보자도 사용할 수 있나요?

네. 어려운 토지 용어와 행정 정보를 쉽게 이해할 수 있도록 요약 설명을 함께 제공합니다.

### 어떤 지역을 볼 수 있나요?

서비스 출시 범위에 따라 제공 지역이 달라질 수 있습니다. 초기에는 데이터 확보와 검증이 가능한 지역부터 순차적으로 제공하는 것을 권장합니다.
